## 🔧 Issues Fixed

### 1. Authorization Test Fixture Issue ✅ Fixed

**Problem:** TestAuthHandler generated new random Guid for `sub` claim on every HTTP request, causing authorization failures for multi-request scenarios (create → edit/delete).

**Root Cause:** In `BackofficeTestFixture.cs` (line 165-166 before fix):
```csharp
new Claim(ClaimTypes.NameIdentifier, Guid.NewGuid().ToString()),
new Claim("sub", Guid.NewGuid().ToString()),
```

**Solution:** Implemented stable admin user ID pattern using dependency injection:

1. Added `TestAdminUserId` constant: `00000000-0000-0000-0000-000000000001`
2. Created `ITestAuthContext` interface and `TestAuthContext` implementation
3. Modified `TestAuthHandler` to accept `ITestAuthContext` via constructor DI
4. Modified `TestAuthHandler.HandleAuthenticateAsync()` to use stable admin user ID

**File Modified:** `tests/Backoffice/Backoffice.Api.IntegrationTests/BackofficeTestFixture.cs`

**Result:** All 3 previously failing authorization tests now pass:
- ✅ `EditOrderNote_WithValidData_UpdatesNoteTextAndReturns204`
- ✅ `DeleteOrderNote_WithValidNote_SoftDeletesNoteAndReturns204`
- ✅ `DeleteOrderNote_WithAlreadyDeletedNote_Returns400`

## 🎓 Key Learnings

### Test Fixture Patterns
1. **Stable Identity Required:** Multi-request scenarios (CRUD workflows) require stable user IDs across requests
2. **DI for Test Context:** Use interfaces + DI to inject test configuration into authentication handlers
3. **Parallel Patterns:** Compare with other test fixtures (Shopping, Orders) to identify missing helpers

### Boundary Testing
1. **Always Test Exact Boundary:** 2000-character limit requires tests for 1999, 2000, and 2001 characters
2. **Boundary Tests Catch Bugs:** Off-by-one errors in validation logic (e.g., `>` vs `>=`)

### Unit vs Integration Split
1. **Unit Tests:** Pure functions (validators, Apply methods) - fast, no infrastructure
2. **Integration Tests:** HTTP endpoints, persistence, authorization - slower, real infrastructure
3. **Coverage Overlap:** Validation tested at both levels (unit for logic, integration for HTTP 400 responses)
