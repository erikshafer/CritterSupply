
**File:** `src/Backoffice Identity/BackofficeIdentity/Authentication/Login.cs`

## 📚 Lessons Learned

### 1. Consistent Role String Formatting is Critical

**Problem:** Using `ToString()` on enums produces PascalCase, but authorization policies expect kebab-case.

**Pattern:**
```csharp
// ❌ WRONG - PascalCase
user.Role.ToString()  // "SystemAdmin"

// ✅ RIGHT - Kebab-case
user.Role.ToRoleString()  // "system-admin"
```

**Why It Matters:** ASP.NET Core authorization is case-sensitive. JWT claims MUST match policy role names exactly.

---

### 2. Test-ID Naming Conventions Must Be Documented

**Problem:** Page Object Models define expected test-ids, but Razor components must actually implement them. Without clear conventions, mismatches occur.

**Proposed Convention:**

| Element Type | Convention | Example |
|--------------|-----------|---------|
| KPI Card | `kpi-{metric-name}` | `kpi-total-orders`, `kpi-revenue` |
| KPI Value (nested) | `kpi-value` | Always `kpi-value` (scoped to parent) |
| Navigation Links | `nav-{destination}` | `nav-customer-service`, `nav-operations` |
| Form Inputs | `{page}-{field}` | `login-email`, `login-password` |
| Buttons | `{page}-{action}` | `login-submit`, `logout-button` |
| Alerts/Banners | `{purpose}-{state}` | `realtime-connected`, `realtime-disconnected` |

**Recommendation:** Document in `docs/skills/e2e-playwright-testing.md` Section: "Test-ID Naming Conventions"

---

### 3. Page Object Models Should Be Written Before Components

**Problem:** Dashboard.razor was written first, then DashboardPage.cs expected different test-ids.

**Better Approach:**
1. Write Gherkin `.feature` file (user stories)
2. Write Page Object Model with expected test-ids
3. Write Razor component implementing those test-ids
4. Write step definitions using Page Object Model

**Why:** POM defines the contract. Components fulfill the contract.

---

### 4. Realtime Indicators Need Both Connected and Disconnected States

**Problem:** Session 13 added `realtime-disconnected` test-id, but no `realtime-connected` test-id existed.

**Solution:** Added `else if (HubService.State == HubConnectionState.Connected)` branch with success banner:

```razor
@if (HubService.State == HubConnectionState.Disconnected)
{
    <MudAlert data-testid="realtime-disconnected">...</MudAlert>
}
else if (HubService.State == HubConnectionState.Connected)
{
    <MudAlert data-testid="realtime-connected">...</MudAlert>
}
```

**Why:** E2E tests need to assert both connected and disconnected states.

---

### 5. Nested Test-IDs Simplify Value Extraction

**Problem:** DashboardPage.cs methods like `GetTotalOrdersValueAsync()` expected nested `[data-testid='kpi-value']` inside KPI cards.

**Solution:** Add `data-testid="kpi-value"` to `<MudText Typo="Typo.h4">` elements:

```razor
<MudPaper data-testid="kpi-total-orders">
    <MudText Typo="Typo.h4" data-testid="kpi-value">@_activeOrders</MudText>
</MudPaper>
```

**Why:** Playwright can scope queries: `TotalOrdersCard.Locator("[data-testid='kpi-value']")`
