### 5. Test-ID Conventions (Session 15) ✅

**Convention Established:**

| Element Type | Pattern | Examples |
|--------------|---------|----------|
| KPI Cards | `kpi-{metric-name}` | `kpi-total-orders`, `kpi-revenue`, `kpi-pending-returns` |
| KPI Values (nested) | `kpi-value` | Always nested within KPI card element |
| Navigation Links | `nav-{destination}` | `nav-customer-service`, `nav-operations`, `nav-analytics` |
| Form Inputs | `{form}-{field}` | `login-email`, `login-password` |
| Form Buttons | `{form}-{action}` | `login-submit`, `logout-button` |
| Real-time Indicators | `realtime-{state}` | `realtime-connected`, `realtime-disconnected` |

**Anti-Patterns to Avoid:**
- ❌ Generic names: `button1`, `div-content`, `label-text`
- ❌ Component-specific names: `mud-button`, `mud-card`
- ❌ Presentational names: `red-banner`, `large-font`
- ❌ Index-based names: `kpi-1`, `alert-2`

**Rationale:**
- Semantic names describe what the element represents, not how it looks
- Kebab-case aligns with HTML attribute conventions
- Hierarchical structure (parent-child) makes locators more robust
- Stable identifiers don't change when UI styling or ordering changes
